## Description

Add offline mode to enable personal ticket tracking in ~/.bodega/<project>/ that acts as a fallback when no local .bodega/ directory exists.

## Goals
- Allow users to track tickets without committing to repository
- Enable consistent tracking across multiple clones of same project
- Maintain simple precedence: local .bodega/ > offline store > not found

## Phases
1. Core Infrastructure (utils, config updates)
2. Init Command Enhancement (--offline flag)
3. Storage Layer Updates & Testing
4. Future: Migration & Sync utilities

## Success Criteria
- `bodega init --offline` creates ~/.bodega/<project>/
- Commands transparently use offline store when no local .bodega/ exists
- Local .bodega/ always takes precedence over offline
- Comprehensive test coverage
